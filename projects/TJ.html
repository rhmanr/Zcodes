Analysis of Travel Patterns and Operational Efficiency of Transjakarta Services Using Machine Learning

Machine Learning, Data Analysis, Data Science Completed: December 2024

Project Highlights

  • Utilized machine learning algorithms to analyze travel patterns of Transjakarta users.
  • Identified key factors affecting operational efficiency and service quality.
  • Developed predictive models to forecast demand and optimize resource allocation.
  • Created interactive dashboards for real-time data visualization and insights.

About the Project

Analysis of Travel Patterns and Operational Efficiency of Transjakarta Services Using Machine Learning is a project aimed at leveraging machine learning techniques to gain insights into the travel behavior of Transjakarta users. By analyzing historical data, the project seeks to identify patterns and trends that can inform decision-making and improve service delivery.

With an interactive web interface built using Streamlit, the system is designed to be accessible, lightweight, and easy to use for students, counselors, or institutions aiming to perform early screening of emotional and psychological conditions among students adapting to a new environment.

Technical Implementation

The system is built using Python and implements various machine learning algorithms to analyze travel data. Here's a breakdown of its main components:

  • Data Preprocessing: Involves cleaning, handling missing values, and formatting raw travel data (tap-in/tap-out logs, timestamps, route codes) for consistent analysis
  • Feature Engineering: Derives informative features such as average trip duration, peak hour usage, passenger density, and route occupancy rates to enhance model performance
  • Model Training: Implements machine learning algorithms such as Random Forest and Logistic Regression to classify route efficiency and detect travel patterns
  • Visualization: Utilizes Matplotlib, Seaborn, and Plotly to generate visual insights, including line charts, heatmaps, and correlation graphs

Challenges & Solutions

One major challenge was dealing with incomplete and noisy real-world data, which often included missing tap-in/tap-out logs or inconsistent timestamps. This was handled through data cleaning, imputation, and validation steps. Another challenge was identifying relevant features that truly impact route efficiency and passenger behavior. Through iterative experimentation, the model was improved by adding engineered features such as average travel time, boarding volume, and peak hour ratios.
